Synty Studios Assets and Mixamo Animation Integration Guide

UModeler X enables the use of Synty Studios' diverse 3D assets and Mixamo's animations within Unity.

Previously, the process involved exporting 3D characters created by Synty Studios using an FBX Exporter, then uploading to Mixamo to use animations. This guide allows you to freely apply animations without leaving Unity.

The overall workflow is as follows.

  1. Select the Synty Studios assets and proceed with the process of converting them into UModeler X objects.

  2. Next, proceed with rigging and skinning according to Mixamo's bone structure.

  3. Afterwards, download animations from the Mixamo service and import them into your Unity project to apply to the model.

info

Synty Studios Assets‚Äã

Synty Studios is a company that produces and sells high-quality 3D assets for various fields. Particularly popular in the Unity Asset Store, these assets can be quickly imported into Unity projects for use.

Synty Studios Asset Store

Synty Studios' assets mainly consist of 3D backgrounds and character models, and are easily editable through UModeler X, allowing for customization to fit your project.

info

Mixamo Service‚Äã

Mixamo is an online platform for 3D character animation. As an Adobe service, Mixamo particularly simplifies the animation and rigging process, allowing game developers and animators to quickly create and apply character animations without complex technical knowledge.

Mixamo Website

Mixamo's library offers animation clips created with motion capture, allowing for easy exploration and selection of various character actions.

Using Mixamo's automatic rigging feature, you can immediately apply animations to your desired character models, significantly reducing project workload.

1. Synty Studios Asset Conversion‚Äã

The first step in the conversion process is to transform the chosen Synty Studios assets into UModeler X objects.

  1. Import Synty Studios assets into your Unity project. Once imported, you will have access to the Prefabs folder.
  2. In the Prefabs folder, find the character prefab you wish to convert and drag and drop it into the scene.
  3. In the scene-placed prefab, select the object with the Skinned Mesh Renderer or Mesh Renderer component. Only objects with meshes can be converted in UModeler X.
info

Static and Skinned Meshes‚Äã

When you create and use 3D models in Unity, you use two types of meshes.

Static Meshes: A Static Mesh is an object that does not deform, meaning it is a mesh that does not animate. They are primarily used for non-moving objects such as backgrounds, buildings, and furniture.

  • Of course, the "universal" use of Static Meshes is for furniture and buildings, but if you need deformation within the object, you can create a Skinned Mesh.

Skinned Meshes: A Skinned Mesh is a 3D model that can be rigged and animated. They are used for moving objects such as characters and animals. Skinned meshes are connected so that their vertices move using bones and weight values, allowing them to be animated.

tip

How to identify Static and Skinned Meshes in Unity‚Äã

  1. select your 3D model.
  2. When selected, the Inspector window displays the model's information.
    • If you have a Mesh Filter component and a Mesh Renderer component:
      1. This is a Static Mesh.
      2. These two components are responsible for the mesh's information and settings for rendering.
    • If you have a Skinned Mesh Renderer component:
      1. This is a Skinned Mesh.
      2. This component supports animation and has the ability to deform the mesh based on bone and weight values.
  1. In the Hierarchy tab, right-click the selected object and choose UModeler X > UModelerize Merge Vertices.
    • UModelerize Merge Vertices is used when you want to combine vertices located in the same position during the UModeler X object conversion.
    • Synty Studios' low-polygon style assets are designed with separated faces, so a process of combining vertices is necessary.
    • UModeler X maintains the hard edge state even after vertices are combined through UModelerize Merge Vertices.
caution

Solving Scale Issues Before Animation Application‚Äã

Before rigging and applying animations, it's crucial to check the character's transform values, especially the scale value.

Occasionally, when checking the UModelerize results, you might find the scale value is not 1. This situation occurs when the object's scale was not 1 before UModelerize. Applying Mixamo animation in this state may lead to the character suddenly enlarging or shrinking during animation playback.

Resetting Scale Values‚Äã

  1. Check the transform values of the character, either before conversion or after becoming a UModeler X object, to ensure the scale is 1.
  2. If the scale value is not 1, select the Reset X Form tool from the Misc group in UModeler X's modeling mode.
  3. After selecting the Reset X Form tool, click the Reset Scale button to reset the scale value to 1.

3. Using Mixamo Bone Structure‚Äã

To apply Mixamo animation to Synty Studios character assets, you need to match the bone structure.

  1. Access the Mixamo website and select Animations from the top category.
    • If you are a first-time user of Mixamo, you can view the T-pose animation.
    • If you have used Mixamo before, type 'T-pose' in the search bar and select the T-Pose animation.
  2. Click the Download button, and in the download options screen, click the Download button again to download the animation file.
  3. Import the downloaded fbx file into your Unity project.
  4. Drag the file into the scene, and in the Hierarchy tab, copy and separate the sub-object 'mixamorig:Hips'.
  5. Separately handle the mixamorig:Hips object, and delete the model object.
    • This bone structure needs to be applied to the UModelerize Synty Studios character assets.
  6. Register the bones as sub-objects of the UModeler X object.
    • Delete the existing Root objects and assign the mixamorig:Hips object.
  7. In the rigging mode, drag the mixamorig:Hips object to the Root Bone slot and click the Remap Rigging button to proceed with rigging.
  8. Depending on the character, you may need to adjust the position of the bones.
    • Bones can be selected from the Bone Tree area in rigging mode,
    • Or, with the Enable Clicking On Bones feature activated, you can directly select bones in the scene by Shift-clicking on them.
  9. Proceed with the skinning process, which connects the vertices of the polygons to the bones.
    • Even after adjusting the bone positions to match the character's shape, the actual vertices and bones are not connected. In this case, the vertices do not follow the moving bones, and you must undergo a process called skinning.
    • Skinning is the process of connecting the surface vertices to the bones, allowing the polygons to move along with the bones when they move. You can use the Auto Weight tool or other skinning-related tools.

4. Applying Mixamo Animation‚Äã

Once the rigging mode process of setting up the bones and skinning for the character is completed, it's time to apply the animation.

  1. Access the Mixamo website and select the desired animation, then click the Download button.
  2. In the Download Settings window, adjust the following settings:
    • Format: Set to FBX Binary (.fbx) format.
    • Skin: Select 'Without Skin' to only get the animation.
      • Since rigging has already been completed with UModeler X, the modeling is not needed.
    • Frames per Second: Represents the number of frames per second, typically set to 30.
    • Keyframe Reduction:
      • The animations provided by Mixamo service are composed of motion capture data, which includes numerous animation key points for each frame.
      • Keyframe Reduction is a feature that reduces the number of key points in the animation, simplifying the data for easier management of the animation.
      • However, sometimes the animation can become slightly distorted when reducing keyframe numbers.
  3. After adjusting the settings, click the Download button to download the animation file and import it into your Unity project.
  4. Select the character and add the Animation component in the Inspector tab.
  5. Drag and drop the downloaded and imported animation file into the Animation component. ![Alt text](img/Mixamo_

ani_04.png)

  • The animation file is created under the imported FBX with the name 'mixamo.com'.
  1. Open the Animation window by clicking Window > Animation > Animation or pressing Ctrl+6.
  2. Press the play button in the Animation window to start the animation playback.
    • If the bones and vertices do not move as desired, you may need to adjust them using weight-related tools.
    • If it's difficult to edit during animation playback, deactivate the Preview button in the upper left of the Animation window to return to the T-pose state, allowing for easier editing.
