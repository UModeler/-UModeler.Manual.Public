트러블 슈팅

UModeler Hub​

회원 가입이 안됩니다.​

  • Confict(409) 에러가 발생합니다.

    • 이메일이나 Username이 이미 다른 사용자에 의해서 사용중일 때 발생합니다. 다른 이메일 주소나 Username을 사용해보세요.

  • 가입하는 도중 'An error occurred while sending the request'라는 오류 메시지를 받았습니다. 이것은 무엇을 의미하며 어떻게 해결할 수 있나요?

    이 오류는 일반적으로 요청을 보내는 시스템과 이를 받아야 할 서버 간의 통신 과정에 문제가 있음을 나타냅니다. 일반적인 원인은 다음과 같습니다:

    • 네트워크 문제: 불안정하거나 나쁜 인터넷 연결이 데이터 전송을 방해할 수 있습니다.

    • 방화벽 또는 보안 설정: 때때로 방화벽이나 보안 설정이 요청을 차단할 수 있습니다.

    • 해결 방법:

      • 인터넷 연결 확인: 인터넷 연결이 안정적인지 확인하세요.
      • 기다렸다가 다시 시도하기: 몇 분 기다린 후 다시 시도하면 서버의 일시적인 과부하나 다운타임과 관련된 문제를 해결할 수 있습니다.
      • 시스템 설정 확인: 방화벽과 보안 설정이 요청을 차단하지 않는지 확인하세요.
      • 지원팀에 문의하기: 문제가 지속될 경우, 사용 중인 서비스의 지원팀에 특정 도움을 요청하세요.
      • 지원팀 이메일 : contact.support@umodeler.com

      기억하세요, 정확한 해결책은 오류의 특정 상황에 따라 다를 수 있습니다.

  • 인증 메일이 도착하지 않았고 Not Acceptable(406) 에러가 발생합니다.

    안티바이러스 방화벽 프로그램이 실행되는지 확인해주세요. 혹시 실행되고 있다면 회원 가입 과정에서는 이 방화벽 프로그램을 비활성화 시키고 시도해보시기 바랍니다.

  • 정상적인 이메일인가요?

    우리 서비스가 발송하는 이메일을 수신한 사용자만 서비스를 이용할 수 있습니다. 간혹 이메일 서비스 제공자에 의해 우리가 발송한 이메일의 수신이 차단될 수 있습니다.

  • 스팸함에 이메일이 도착했나요?

    우리 서비스가 발송한 이메일이 스팸함에 있는지 확인해 보세요. 우리 서비스는 사용자가 이메일을 제대로 수신할 수 없는 상태인 경우 더이상 이메일을 발송하지 않습니다. 만약 이메일이 스팸함에 도착했다면 더 이상 스팸으로 분류되지 않도록 조치를 취해 주세요.

로그인이 안됩니다.​

  • 회원 가입 후 발송되는 인증(Verification) 이메일을 확인하세요.

    혹시 아래와 같은 페이지가 나오면서 로그인이 되지 않나요?

    • 유효하지 않은 이메일 페이지 화면

    그렇다면 아이디로 사용하는 이메일 주소로 인증 메일이 왔는지 확인하세요. 만약 오지 않았다면 스팸함에 있는지 확인하고 인증 과정을 마무리 하세요. 스팸함에도 인증 메일이 없다면 이메일 사업자에 의해서 차단 되었을 가능성이 있습니다. 담당자에게 문의하시기 바랍니다.

  • 인터넷 접속이 원활한지 확인 하십시오.

    로그인 기능은 인터넷 접속이 가능한 환경에서 로그인이 가능합니다.

  • 기업 내 인터넷 환경 또는 HTTP Proxy 환경인지 확인 하십시오.

    일부 기업 내 인터넷 환경인 경우 우리 서비스에 접근하기 위해 네트워크 관리자에게 도메인 접속을 할 수 있도록 요청하세요. 다만, HTTP Proxy 환경인 경우 우리 서비스는 아직 HTTP Proxy 설정을 지원하지 않습니다. 빠른 시일내에 HTTP Proxy 설정 기능을 지원할 예정입니다.

UModeler X Plus의 잠금 해제 페이지에서 인보이스 번호(Invoice No) 또는 주문 ID(Order ID)를 입력하면 Internal Server Error (500) 에러가 발생합니다.​

이 문제는 보통 허브에 이미 로그인된 경우에 발생합니다. 로그아웃한 후 다시 로그인하면 해결될 수 있습니다.

아래와 같이 Sign Out 버튼을 클릭하여 UModeler Hub에서 로그 아웃할 수 있습니다.

패키지 다운로드가 안됩니다.​

  • 인증된 사용자인가요?

    패키지 다운로드는 인증된 사용자에게만 제공됩니다. 로그인 가능한 계정으로만 패키지를 다운로드 할 수 있습니다. 계속 문제가 발생한다면 인증이 만료되었을 수 있습니다. 로그아웃 후 다시 로그인하여 확인하십시오.

패키지 다운로드가 너무 오래 걸립니다.​

  • 제공받는 인터넷 서비스의 품질 확인이 필요합니다.

    우리 서비스는 전세계 모든 국가에서 안정적인 패키지 다운로드 속도를 제공하기 위해 CDN 으로 지리적으로 가장 원활한 서버로부터 패키지를 다운로드 합니다. 일반적으로 사용자의 인터넷 대역폭 또는 속도의 제한으로 다운로드가 오래 걸릴 수 있습니다. 이 경우 인터넷 서비스 사업자에게 문의 하십시오.

서비스 접속이 안되는 것 같습니다.​

  • 기업 인터넷 환경이나 HTTP Proxy 설정이 필요한가요?

    일부 기업 내 인터넷 환경인 경우 우리 서비스에 접근하기 위해 네트워크 관리자에게 문의하세요. 우리 서비스는 아직 HTTP Proxy 설정을 지원하지 않습니다. 빠른 시일내에 HTTP Proxy 를 지원할 예정입니다.

그래도 문제가 계속 발생합니다.​

발생하는 문제에 대해 조치할 수 있도록 contact.support@umodeler.com 이메일 문의를 주시거나 UModeler X 디스코드 서버 이슈 포럼(링크)에 문의 주세요. 가능한한 빨리 문제가 해결될 수 있도록 도와 드리겠습니다.

계정 생성 과정에서 “Illegal byte sequence encounted in the input. Parameter name: string” 에러가 발생합니다.​

이 에러는 아래의 항목에 영어가 아닌 문자열이 포함되어 있을때 발생하는 것으로 파악되었습니다.

  • 유니티 프로젝트 경로
  • 윈도우즈의 컴퓨터 이름(PC or Computer name)

컴퓨터 이름의 경우 커맨드창에서 hostname을 입력하면 아래와 같이 확인할 수 있습니다.

컴퓨터 이름을 변경하는 방법은 윈도우즈 설정창에서 상단의 이름 바꾸기 기능으로 변경할 수 있습니다. 설정창은 윈도우 버튼을 누르고 설정을 입력하면 띄울 수 있습니다.

위의 항목의 이름을 모두 영어로 변경한 후 다시 시도해보시기 바랍니다.

이 이슈에 대한 참고 페이지 입니다.

  • https://github.com/mono/mono/issues/7117
  • https://github.com/BepInEx/BepInEx/issues/407
  • 윈도우11에서 PC 이름을 변경하는 법

UModeler X​

Unity 에디터가 오류 대화상자와 함께 종료되었고, 그 메시지는 다음과 같습니다: “Failed to present D3D11 swapchain due to device reset/removed…​

참고

이 오류는 UModeler X 버전 0.17.3부터 해결되었습니다. 만약 이 문제가 발생했다면 UModeler X를 최신 버전으로 업데이트 해주시기 바랍니다.

UModeler X를 사용하다 보면 가끔씩 아래의 에러 윈도우가 뜨면서 유니티가 종료되는 경우가 발생합니다.

이 문제는 GPU 처리 지연과 관련된 이슈이기 때문에 UModeler X를 사용하지 않는 상태에서도 발생을 하곤합니다. 보통 설치된 그래픽 카드 드라이버를 최신 버전으로 업그레이드를 하면 해결이 될 때도 있습니다.

아래는 이와 관련된 일반적인 해법에 대한 가이드 입니다.

  • Unity Forum : Failed to present D3D11 swapchain due to device reset/removed. List of Solutions
  • Microsoft : Timeout detection and recovery (TDR)

그리고 GPU 메모리, GPU 성능 등의 하드웨어 스펙이 낮을수록 자주 발생하는 경향이 있습니다.

만약 이슈가 계속 발생한다면 UModeler X 0.17.1 버전부터 추가된 Disable GPU Options 속성을 켜고 사용해보세요. 이 토글 속성은 Preferences 윈도우의 Debug 그룹에서 찾으실 수 있습니다.

Disable GPU Options을 켤 경우 Project Settings 윈도우의 Player 옵션에 있는 GPU Skinning과 Use DXGI flip model swapchain D3D11 속성을 비활성화 시켜서 이 오류가 발생할 수 있는 확률을 줄여줍니다.

Project Settings > Player > GPU Skinning

*Project Setting > Player > Use DXGI flip model swapchain D3D11*

Project Setting 에서 Quality 를 Very Low로 바꾸는것도 오류가 발생할 확률을 줄여줍니다.

Project Setting > Quality > Very Low

ProBuilder 툴바와 UModeler X 툴바가 겹쳐서 보여요.​

UModeler X 0.14.7 버전부터 툴바의 위치를 변경할 수 있는 기능이 추가되었습니다.

툴바 위치를 변경하는 방법은 아래와 같습니다.

  1. 툴바의 톱니바퀴 아이콘을 클릭합니다.

  2. Preference 창이 열리면 UModeler X > Display > Toolbar Position 에서 위치를 조정합니다. 앵커의 위치는 중앙 상단으로 되어 있습니다. Y 값을 늘리면 툴바가 아래도 움직이는 것을 확인할 수 있습니다.

HDRP, URP 씬에서 텍스처가 자홍색(Magenta)으로 나와요.​

HDRP 및 URP 프로젝트에 UModeler X를 설치할 경우 기본 머티리얼이 BuiltIn-RP 용 머티리얼로 셋팅되어 있어서 UModeler X 오브젝트의 텍스처가 자홍색으로 보이는 현상이 발생합니다.

이 경우 아래 영상에서와 같이 Preferences의 UModeler X 셋팅에서 HDRP 혹은 URP 머티리얼을 기본 머티리얼로 셋팅을 해주면 해결할 수 있습니다.

이미 자홍색 텍스처가 입혀진 채로 생성된 UModeler X 오브젝트의 경우 Material 툴에서 머티리얼을 HDRP 혹은 URP 머티리얼로 변경해주시기 바랍니다.

Scene View에서 Toolbar가 보이지 않습니다.​

Enter Play Mode Settings 옵션이 활성화되어 있는 경우에 Scene View에서 아래와 같은 Toolbar가 보이지 않을 수 있습니다.

Project Settings > Editor > Enter Play Mode Settings > Enter Play Mode Options 옵션을 비활성화하거나, 해당 옵션을 활성화한 상태에서 Reload Domain 옵션을 활성화하여 해결할 수 있습니다.

UModeler X | 모델링 모드​

인스펙터 윈도우에 [Please Select a UModeler X object to activate this window] 경고 메세지가 표시되면서 UModeler X 관련 UI가 표시되지 않습니다.​

이 메세지는 UModeler X 컴포넌트를 포함하는 게임 오브젝트 혹은 UModeler X 컴포넌트 자체가 비활성화 되었을 때 표시 되는 것이 일반적입니다.

만약 게임 오브젝트와 컴포넌트 모두 활성화 되어 있는데 여전히 이 경고가 표시된다면 설치된 제페토 스튜디오 버전이 3.2.5 혹은 그 이후 버전인지 확인하세요. 3.2.4 버전 혹은 그 이하 버전이 설치된 프로젝트에서도 이 경고가 발생할 수도 있습니다.

UModeler X | 리깅 모드​

리깅 모드에서 [GPU Skinning] 관련 메세지가 표시됩니다.​

현재 [GPU Skinning] 옵션이 켜진 상태에서 리깅 모드를 사용하면 간혹가다가 Direct X 관련 오류가 발생해서 유니티가 다운되는 현상이 발생합니다.

[상단메뉴 Edit → Project Settings 윈도우]에서 [Player → Other Settings] 그룹에 위치한 [GPU Skinning] 옵션을 끄면 안정적인 환경에서 UModeler X 리깅 모드를 사용할 수 있습니다.

이 오류는 추후에 수정될 예정입니다.

리깅 모드 진입할 때 [Set the Root Bone] 메세지가 표시됩니다.​

루트 본(Root Bone)이 설정되지 않은 상태에서 UModeler X 오브젝트의 리깅 모드를 시작하면, 경고 메시지가 나타납니다.

이 문제를 해결하려면, 루트 본이 되는 게임 오브젝트를 하이어라키 윈도우에서 클릭드래그하여 Root Bone 오브젝트 속성 필드에 놓거나, 오른쪽의 디스크 아이콘을 클릭하여 루트 본 오브젝트를 선택하면 됩니다.

일반적으로 루트 본 오브젝트는 여러 하위 본 오브젝트들과 함께 계층 구조를 이루며, 해당 게임 오브젝트의 자식 오브젝트로 존재하는 경우가 자주 있습니다.

리깅 모드에서 트리본 선택의 언두/리두가 제대로 갱신이 안됩니다.​

이 경우 다른 본을 선택했다가 다시 선택하면 갱신이 됩니다.

리깅 모드에 진입할 때 [Rigging bones have been changed. Please proceed with remapping.] 메세지가 표시됩니다.​

등록된 리깅 본 데이터와 실제 Hierachy에 있는 본 트리 데이터가 일치하지 않는 경우 발생합니다. 이때 [Remap Rigging] 버튼을 눌러주시면 현재 Hierachy에 있는 본 트리 데이터로 변환작업을 진행합니다. 기존 BoneWeight 정보의 경우는 RiggingBone 정보의 손상이 없는 경우에 한에서 자동으로 변환을 합니다.

일치하지 않는 경우는 다음과 같습니다.

  • 루트 본의 변경
  • 루트 본 삭제
  • 루트 본 이름 변경
  • 루트 본 위치 변경
  • 기존 자식 본 이름 변경
  • 기존 자식 본 삭제
  • 기존 자식 본 위치 변경
  • 새로운 자식 본이 생성

리깅 모드로 들어갈 때, [The modifiers have to be collapsed.]라는 경고가 나타납니다.​

리깅 모드를 활성화할 때는 모디파이어 스택에 모디파이어가 없어야 합니다. 따라서 리깅 모드로 들어가기 전에 모든 모디파이어를 접거나(Collapse) 제거해야 합니다. 모디파이어에 의해 만들어진 변경 사항을 유지하려면 모디파이어 접기 작업을 수행해야 합니다. 아래 절차에 따라 모디파이러를 접어주세요.

  1. 모디파이어 스택에서 가장 위에 있는 모디파이어를 선택합니다.
  2. 모디파이어 접기(Collapse Modifiers) 버튼을 클릭합니다. 이 작업은 모든 모디파이어를 접고 그들에 의해 만들어진 변경 사항을 원본 메시에 적용합니다.

SkinnedMeshRenderer 를 UModelerize할 때 스키닝 및 뼈대 정보가 제대로 나오지 않습니다.​

  • Root Bone이 제대로 설정이 되어있나 확인해주세요.
  • Root Bone이 설정이 안되어 있고 Hierarchy 윈도우에서 본 계층구조가 보이지 않을 경우 Animator Component에서 본 계층구조가 최적화 되어 있는 경우 입니다. 이 경우 최적화된 본 구조를 Hierarchy 윈도우에 다시 보이도록 하는 작업(Deoptimize)이 필요합니다. 이 절차 대한 설명은 아래을 참고하세요.
    • 해당 기능은 Prefab Instance에서는 작동하지 않으므로 Prefab Instance라면 하기 전에 Hierarchy에서 Unpack을 진행해주세요.
    • SkinnedMeshRenderer를 포함하고 있는 Animator Component에서 Deoptimize Transform Hierarchy를 실행해주세요.
    • 다시 SkinnedMeshRenderer로 돌아와 RootBone이 null이 아닌 GameObject로 선택 되어있는지 확인 후 UModelerize를 진행해주세요.
